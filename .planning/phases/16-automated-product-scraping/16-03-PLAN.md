<objective>
Build the SQLite-based change detection engine that hashes scraped product data and identifies new, changed, and removed products between scrape runs.

Purpose: Avoid re-processing unchanged products on every nightly run. By comparing content hashes, we only sync products that have actually changed, saving WooCommerce API calls, embedding generation time, and Qdrant writes.
Output: ChangeDetectionService with SQLite persistence, content hashing, and diff calculation (new/changed/removed).
</objective>

    - `async def initialize(self)`:
      1. Ensure parent directory exists (os.makedirs with exist_ok=True)
      2. Connect to SQLite via aiosqlite
      3. Create table if not exists:
         ```sql
         CREATE TABLE IF NOT EXISTS product_hashes (
           store_id TEXT NOT NULL,
           external_id TEXT NOT NULL,
           url TEXT NOT NULL,
           content_hash TEXT NOT NULL,
           title TEXT,
           price TEXT,
           image_url TEXT,
           first_seen_at TEXT NOT NULL DEFAULT (datetime('now')),
           last_seen_at TEXT NOT NULL DEFAULT (datetime('now')),
           last_changed_at TEXT NOT NULL DEFAULT (datetime('now')),
           PRIMARY KEY (store_id, external_id)
         );
         ```
      4. Create index on (store_id) for fast per-store queries
      5. Close connection (each method opens its own connection for simplicity with async)

    - `@staticmethod def compute_hash(product: ScrapedProduct) -> str`:

      2. Exclude scraped_at and store_id (these change every run but don't indicate product changes)
      3. Use hashlib.sha256(canonical_string.encode()).hexdigest()
      4. This is a pure function — no async, no side effects

<task type="auto">
  <name>Task 2: Implement diff calculation — detect new, changed, and removed products</name>
  <files>apps/backend/src/features/scraper/service/change_detection.py</files>
  <action>
    Add diff calculation methods to ChangeDetectionService:

    - `async def detect_changes(self, store_id: str, products: list[ScrapedProduct]) -> ChangeReport`:
      The core diffing method:
      1. Open aiosqlite connection to self.db_path
      2. Fetch all existing hashes for this store_id: SELECT external_id, content_hash FROM product_hashes WHERE store_id = ?
      3. Build dict: existing = {external_id: content_hash}
      4. For each scraped product:
         - Compute hash via compute_hash(product)
         - If external_id not in existing → classify as "new"
         - If external_id in existing AND hash differs → classify as "changed"
         - If external_id in existing AND hash matches → classify as "unchanged"
      5. Products in existing but NOT in scraped list → classify as "removed"
      6. Return ChangeReport dataclass
      7. Close connection

    - `async def update_hashes(self, store_id: str, products: list[ScrapedProduct])`:
      After successful sync, update the SQLite database:
      1. Open aiosqlite connection
      2. For each product, UPSERT (INSERT OR REPLACE):
         - store_id, external_id, url, content_hash (computed), title, price, image_url (first), last_seen_at = now
         - For new products: first_seen_at = now, last_changed_at = now
         - For changed products: last_changed_at = now (preserve first_seen_at)
         Use INSERT ... ON CONFLICT(store_id, external_id) DO UPDATE SET ...
      3. Commit and close

    - `async def mark_removed(self, store_id: str, external_ids: list[str])`:
      Delete removed products from the hash table:
      DELETE FROM product_hashes WHERE store_id = ? AND external_id IN (...)
      Use parameterized queries (NOT string interpolation — SQL injection prevention).

    - `async def get_store_stats(self, store_id: str) -> dict`:
      Return {total: int, oldest: str, newest: str} for monitoring.

    Define **ChangeReport** as a dataclass in the same file:
    ```python
    @dataclass
    class ChangeReport:
        store_id: str
        new: list[ScrapedProduct]
        changed: list[ScrapedProduct]
        removed_ids: list[str]  # external_ids no longer in sitemap
        unchanged_count: int
    ```

    Important: All SQL queries MUST use parameterized queries (? placeholders) — never f-strings or string concatenation for SQL.
  </action>
