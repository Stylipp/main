<task type="auto">
  <name>Task 1: Create per-site scrape-and-sync pipeline orchestrator</name>
  <files>apps/backend/src/features/scraper/service/orchestrator.py</files>
  <action>
    Create ScraperOrchestrator that coordinates the full pipeline for a single store, then orchestrates multiple stores.

    Class **ScraperOrchestrator**:

    - `__init__(self, settings: Settings)`:
      Store settings. Services will be initialized in run methods (they need async setup).

    - `async def run_store(self, store: StoreConfig) -> ScrapeResult`:
      Execute the full pipeline for one store:

      1. **Initialize services:**
         - SitemapService()
         - ProductScraper()
         - ChangeDetectionService(settings.scraper_sqlite_path)
         - await change_detection.initialize()

      2. **Fetch sitemap:**
         - urls = await sitemap_service.fetch_product_urls(store)
         - Log: "Found {len(urls)} product URLs for {store.name}"
         - If no URLs: return early with empty ScrapeResult

      3. **Scrape products:**
         - products = await product_scraper.scrape_batch(urls, store)
         - Log: "Scraped {len(products)}/{len(urls)} products from {store.name}"

      4. **Detect changes:**
         - report = await change_detection.detect_changes(store.id, products)
         - Log: "Changes for {store.name}: {len(report.new)} new, {len(report.changed)} changed, {len(report.removed_ids)} removed, {report.unchanged_count} unchanged"

      5. **Sync to systems** (only if there are changes):
         If report.new or report.changed or report.removed_ids:
           a. Open async database session (from core.database)
           b. DbSyncService: sync new and changed products to PostgreSQL
           c. WooCommerceClient: batch create/update to WooCommerce (wrap in try/except — non-fatal)
           d. VectorSyncService: embed and upsert new/changed products to Qdrant
           e. Commit database session
           f. Update change detection hashes: await change_detection.update_hashes(store.id, report.new + report.changed)
           g. If removed: await change_detection.mark_removed(store.id, report.removed_ids)

      6. **Build and return ScrapeResult** with all counts and timing

      Error handling: Wrap entire store pipeline in try/except. On failure, log the error and return a ScrapeResult with the error message. Never let one store failure crash the multi-store run.

    - `async def run_all(self, stores: list[StoreConfig] | None = None) -> PipelineResult`:
      Run all enabled stores with per-site parallelism:
      1. Load store configs if not provided: load_store_configs()
      2. Filter to enabled stores only
      3. Log: "Starting scraper pipeline for {n} stores"
      4. Start timer
      5. Run stores concurrently: results = await asyncio.gather(*[self.run_store(store) for store in enabled_stores], return_exceptions=True)
      6. Handle exceptions in results: convert to ScrapeResult with error
      7. **Re-clustering decision:**
         - Sum total new + changed across all stores
         - Get total product count from PostgreSQL (SELECT COUNT)
         - Check VectorSyncService.should_recluster(changed_total, product_total, settings.scraper_recluster_threshold)
         - If yes: trigger re-clustering via VectorSyncService.trigger_recluster(session)
      8. Build and return PipelineResult with timing, all ScrapeResults, SyncResults, recluster status
      9. Log final summary: "Pipeline complete: {total_new} new, {total_changed} changed, {total_removed} removed across {n} stores in {duration}s"

    Import all service types, StoreConfig, ScraperConfig, load_store_configs, schemas, Settings, async database session maker, Qdrant client getter, EmbeddingService, QualityGateService.

<task type="auto">
  <name>Task 2: Create CLI entry point script for nightly execution</name>
  <files>apps/backend/scripts/run_scraper.py</files>
  <action>
    Create a standalone CLI script following the seed_bootstrap.py pattern.

    Script: `apps/backend/scripts/run_scraper.py`

    Usage: `python -m scripts.run_scraper [options]`

    Implementation:

    ```python
    """Automated product scraper — nightly pipeline runner.

    Usage:
        python -m scripts.run_scraper              # Run all enabled stores
        python -m scripts.run_scraper --store mekimi  # Run specific store only
        python -m scripts.run_scraper --dry-run     # Scrape and diff only, no sync
        python -m scripts.run_scraper --list-stores # Show configured stores
    """
    ```

    1. **Argument parsing** (use argparse):
       - `--store STORE_ID`: Run only this specific store (by store config id)
       - `--dry-run`: Scrape and detect changes but don't sync to any system
       - `--list-stores`: Print configured stores and exit
       - `--verbose` / `-v`: Enable DEBUG logging

    2. **Main async function:**
       - Configure logging (INFO by default, DEBUG with --verbose)
       - Load Settings from .env
       - If --list-stores: load configs, print table (id, name, enabled, sitemap_url), exit
       - If --store: filter to that store only
       - Initialize ScraperOrchestrator(settings)
       - If --dry-run: use a modified run that skips sync steps (pass a dry_run flag)
       - Otherwise: await orchestrator.run_all(stores)
       - Print summary table to stdout:
         ```
         ╔══════════════════════════════════════════════╗
         ║          Scraper Pipeline Results            ║
         ╠══════════════════════════════════════════════╣
         ║ Store      │ URLs │ New │ Changed │ Removed  ║
         ║ mekimi     │  422 │  12 │      3  │       0  ║
         ║ exsos      │  315 │   0 │      0  │       2  ║
         ╠══════════════════════════════════════════════╣
         ║ Re-cluster: No (0.4% < 5.0% threshold)      ║
         ║ Duration: 4m 32s                             ║
         ╚══════════════════════════════════════════════╝
         ```
       - Exit code 0 on success, 1 on any store failure

    3. **Entry point:**
       ```python
       if __name__ == "__main__":
           asyncio.run(main())
       ```

    4. **Dry-run support:**
       Add a `dry_run: bool = False` parameter to ScraperOrchestrator.run_store() and run_all().
       When dry_run=True:
       - Execute steps 1-4 (sitemap, scrape, detect changes) normally
       - Skip step 5 (sync to WooCommerce, PostgreSQL, Qdrant)
       - Skip re-clustering
       - Still return full ScrapeResult with change counts
       - Log: "[DRY RUN] Would sync {n} new, {m} changed products"

    Follow the seed_bootstrap.py pattern for:
    - Loading Settings
    - Setting up async database session
    - Loading EmbeddingService model
    - Proper cleanup on exit
  </action>
