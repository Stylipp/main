  <action>
    Create the scraper feature module following the project's feature folder convention (see features/products/ and features/ai/ for pattern).

    In schemas/schemas.py, define these Pydantic models:

    1. **ScrapedProduct** — represents a single product extracted from a store page:
       - url: str (source product page URL)
       - external_id: str (derived from URL hash if no explicit ID — use hashlib.md5(url.encode()).hexdigest()[:16])
       - title: str
       - description: str | None = None
       - price: Decimal | None = None (None if unparseable)
       - sale_price: Decimal | None = None
       - currency: str = "ILS" (Israeli stores default)
       - image_urls: list[str] (all product images found)
       - categories: list[str] = [] (breadcrumb categories)
       - store_id: str (which store config this came from)
       - scraped_at: datetime (when scraped)

    2. **ScrapeResult** — result of scraping a single store:
       - store_id: str
       - total_urls: int (URLs found in sitemap)
       - total_scraped: int (successfully scraped)
       - total_failed: int (scrape errors)
       - total_new: int (new products detected)
       - total_changed: int (changed products detected)
       - total_removed: int (products no longer in sitemap)
       - duration_seconds: float
       - errors: list[str] = [] (error messages for failed URLs)

    3. **SyncResult** — result of syncing changes to backend systems:
       - store_id: str
       - woo_created: int
       - woo_updated: int
       - db_created: int
       - db_updated: int
       - qdrant_upserted: int
       - qdrant_removed: int
       - errors: list[str] = []

    4. **PipelineResult** — overall nightly run result:
       - started_at: datetime
       - finished_at: datetime
       - stores: list[ScrapeResult]
       - sync: list[SyncResult]
       - recluster_triggered: bool

    All __init__.py files should be empty except schemas/__init__.py which re-exports the schema classes.
  </action>
  <verify>python -c "from src.features.scraper.schemas.schemas import ScrapedProduct, ScrapeResult, SyncResult, PipelineResult; print('OK')" from apps/backend directory</verify>
  <done>All schema classes importable, feature folder structure matches project convention</done>
</task>

  <action>
    Create a YAML-based store configuration system that defines which stores to scrape and how.

    In config/stores.yaml, define the initial store configs based on existing Sprapper input files. Each store entry:

    ```yaml
    stores:
      - id: "mekimi"
        name: "Mekimi"
        enabled: true
        sitemap_url: "https://mekimi.co.il/product-sitemap.xml"
        base_url: "https://mekimi.co.il"
        currency: "ILS"
        rate_limit_seconds: 0.5
        timeout_seconds: 30
        product_url_pattern: "/product/"
        woo_store_id: "woo-cloudways"
        selectors:
          title: ".product_title h1, h1.product_title"
          price_regular: ".summary .price del .woocommerce-Price-amount"
          price_sale: ".summary .price ins .woocommerce-Price-amount"
          price_single: ".summary .price .woocommerce-Price-amount"
          description: ".woocommerce-product-details__short-description"
          categories: ".woocommerce-breadcrumb a"
          images: ".woocommerce-product-gallery img[data-large_image]"
          stock_in: ".stock.in-stock"
          stock_out: ".stock.out-of-stock"

      - id: "exsos"
        name: "Exsos"
        enabled: false
        sitemap_url: "https://exsos.co.il/product-sitemap.xml"
        base_url: "https://exsos.co.il"
        currency: "ILS"
        rate_limit_seconds: 0.5
        timeout_seconds: 30
        product_url_pattern: "/product/"
        woo_store_id: "woo-cloudways"
        selectors:
          title: ".product_title h1, h1.product_title"
          price_regular: ".summary .price del .woocommerce-Price-amount"
          price_sale: ".summary .price ins .woocommerce-Price-amount"
          price_single: ".summary .price .woocommerce-Price-amount"
          description: ".woocommerce-product-details__short-description"
          categories: ".woocommerce-breadcrumb a"
          images: ".woocommerce-product-gallery img[data-large_image]"
          stock_in: ".stock.in-stock"
          stock_out: ".stock.out-of-stock"
    ```

    In config/stores.py, create:

    1. **StoreSelectors** (Pydantic BaseModel) — CSS selectors for product extraction:
       - title, price_regular, price_sale, price_single, description, categories, images, stock_in, stock_out: all str

    2. **StoreConfig** (Pydantic BaseModel) — single store configuration:
       - id: str
       - name: str
       - enabled: bool = True
       - sitemap_url: str
       - base_url: str
       - currency: str = "ILS"
       - rate_limit_seconds: float = 0.5
       - timeout_seconds: float = 30.0
       - product_url_pattern: str = "/product/"
       - woo_store_id: str (which WooCommerce store to sync to)
       - selectors: StoreSelectors

    3. **ScraperConfig** (Pydantic BaseModel) — top-level config:
       - stores: list[StoreConfig]

    4. **load_store_configs(config_path: Path | None = None) -> ScraperConfig** function:
       - Default path: Path(__file__).parent / "stores.yaml"
       - Load YAML with PyYAML (already in Python ecosystem; add to pyproject.toml if missing)
       - Parse into ScraperConfig
       - Return only enabled stores when called with a filter

    In core/config.py Settings class, add:
    - scraper_sqlite_path: str = "data/scraper_state.db" (path for SQLite change-tracking DB, relative to backend root)
    - scraper_recluster_threshold: float = 0.05 (re-cluster if >5% products changed)

    Do NOT add PyYAML to pyproject.toml yet — just use it in the import. It will be added as a dependency during execution if missing.
  </action>
