  <behavior>
    **1. compute_freshness_score(created_at: datetime) -> float**
    - 14-day half-life exponential decay: score = exp(-λ * days_old), λ = ln(2)/14
    - Cases:
      - Just created (today) → ~1.0
      - 14 days ago → ~0.5
      - 28 days ago → ~0.25
      - 0 days (same instant) → 1.0
      - Future date → 1.0 (clamp, don't penalize)

    **2. compute_price_affinity(product_price, price_median, price_std) -> float**
    - Log-normal Gaussian: score = exp(-((ln(price) - ln(median))^2) / (2 * ln(sigma)^2))
    - sigma = max(price_std, price_median * 0.3) — prevents zero/tiny sigma
    - Cases:
      - Price exactly at median → ~1.0
      - Price 2x median → score < 1.0 but > 0
      - Price 0.5x median → similar to 2x (symmetric in log space)
      - price <= 0 → 0.5 (neutral fallback)
      - median <= 0 → 0.5 (neutral fallback)

    **3. normalize_scores(scores: list[float]) -> list[float]**
    - Min-max normalization to [0, 1]
    - Cases:
      - [0.2, 0.5, 0.8] → [0.0, 0.5, 1.0]
      - [0.5, 0.5, 0.5] → [0.5, 0.5, 0.5] (uniform → all 0.5)
      - [] → []
      - [0.7] → [0.5] (single element → 0.5)

    **4. rank_candidates(candidates, user_price_profile, cluster_priors) -> list[RankedCandidate]**
    - Input: list of Qdrant ScoredPoint results with payload {product_id, price, created_at, cluster_id}
    - Normalize each factor across the candidate batch
    - Apply weights: 0.65 * cosine + 0.15 * cluster_prior + 0.10 * price_affinity + 0.10 * freshness
    - Return sorted descending by final_score
    - Cases:
      - 3 candidates with known scores → verify correct ordering
      - Empty candidates → empty result
      - All same scores → stable ordering (no crash)
  </behavior>

  <implementation>
    **scoring.py** — Pure functions: compute_freshness_score, compute_price_affinity, normalize_scores.
    Uses math stdlib for exp/log, datetime for time delta. No external dependencies beyond stdlib.

    **ranking_service.py** — RankedCandidate dataclass (product_id, score, cosine_score, cluster_prior_score, price_score, freshness_score). rank_candidates function that:
    1. Extracts raw scores from each factor
    2. Normalizes each factor array via normalize_scores
    3. Applies weighted combination
    4. Sorts and returns RankedCandidate list

    Follow RESEARCH.md code examples for freshness (λ = ln(2)/14) and price affinity (log-normal Gaussian).
    Do NOT hand-roll vector search or score normalization beyond simple min-max — per RESEARCH.md dont_hand_roll section.
