<task type="auto">
  <name>Task 1: Create SwipeCard component with Motion drag gestures</name>
  <files>apps/stylipp.com/src/features/feed/components/SwipeCard.tsx</files>
  <action>
Build SwipeCard as the core draggable product card component.

**Props interface:**
- item: FeedItem — product data to display
- isTop: boolean — only the top card is draggable
- onSwipe: (direction: SwipeDirection) => void — callback when swipe threshold met
- exitX: number — x position for exit animation (set by parent before unmount)

**Gesture physics (from 6-RESEARCH.md):**
- Use `useMotionValue(0)` for x position tracking
- `useTransform(x, [-200, 200], [-15, 15])` for card rotation during drag
- `drag="x"` for horizontal-only dragging
- `dragSnapToOrigin` to return card to center if threshold not met
- `dragElastic={0.7}` for resistance feel
- `dragTransition={{ bounceStiffness: 300, bounceDamping: 20 }}` for snap-back spring
- `style={{ touchAction: 'none' }}` on the draggable element — CRITICAL for mobile

**onDragEnd handler:**
- Check `info.offset.x > 100 || info.velocity.x > 500` → swipe right (LIKE)
- Check `info.offset.x < -100 || info.velocity.x < -500` → swipe left (DISLIKE)
- If neither threshold met, dragSnapToOrigin handles return

**Card content (adapt CalibrationStep pattern):**
- Product image (full card background or CardMedia, aspect-ratio for consistent sizing)
- Product title (Typography)
- Price with currency (Typography)
- Explanation text from feed (Typography, subtle)
- Product URL link (small external link icon) — but use onPointerDownCapture with e.stopPropagation() to prevent drag interference on the link

**Exit animation:**
- `exit={{ x: exitX, opacity: 0, rotate: exitX > 0 ? 20 : -20, transition: { duration: 0.3 } }}`

**Styling:**
- Use MUI theme colors (background paper, text primary/secondary)
- Card should be ~90vw max-width on mobile, ~400px on desktop
- Border-radius: 16px for modern card feel
- Box shadow matching theme card shadow pattern
- `whileDrag={{ cursor: 'grabbing', scale: 1.02 }}` for tactile feedback
- Set `pointerEvents: isTop ? 'auto' : 'none'` to prevent interaction on stacked cards below

Do NOT add save gesture (swipe up) — save is button-only per research recommendation.

**SwipeIndicators.tsx:**
Visual overlays that appear on the card during drag to show intent.

- Accept `x: MotionValue<number>` prop from parent SwipeCard
- Use `useTransform(x, [0, 100], [0, 1])` for LIKE indicator opacity
- Use `useTransform(x, [-100, 0], [1, 0])` for DISLIKE indicator opacity
- LIKE indicator: green border/stamp with "LIKE" text + FavoriteIcon, positioned top-right of card, rotated ~-15deg
- DISLIKE indicator: red border/stamp with "NOPE" text + CloseIcon, positioned top-left of card, rotated ~15deg
- Both are motion.div with `style={{ opacity: likeOpacity/dislikeOpacity }}`
- Use bold, uppercase text with 3px border (stamp aesthetic)
- Position: absolute within the card, z-index above card content

**SwipeActions.tsx:**
Button controls below the card stack for accessibility and non-gesture interaction.

Props:
- onLike: () => void
- onDislike: () => void
- onSave: () => void
- onUndo: () => void
- canUndo: boolean
- disabled: boolean — disable during exit animation

Buttons (left to right):
1. Undo (UndoIcon) — small, subtle, disabled when canUndo=false
2. Dislike (CloseIcon) — medium red circle, prominent
3. Save (BookmarkIcon) — medium secondary color circle
4. Like (FavoriteIcon) — medium green circle, prominent

Use MUI IconButton with custom sx styling:
- Dislike/Like: 56px circles with colored backgrounds (error.main / success.main)
- Save: 48px circle with secondary color
- Undo: 40px circle, outlined variant, muted

All buttons get `whileHover={{ scale: 1.1 }}` and `whileTap={{ scale: 0.9 }}` via motion.create wrapping.
