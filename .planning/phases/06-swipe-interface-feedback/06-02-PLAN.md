**types/swipe.ts:**
Define TypeScript types for the swipe system:
- SwipeDirection: 'left' | 'right' (left = dislike, right = like)
- SwipeAction: 'like' | 'dislike' | 'save'
- FeedItem: { product_id: string, title: string, price: number, currency: string, image_url: string, product_url: string, score: number, explanation: string } — mirrors backend FeedItem schema
- FeedResponse: { items: FeedItem[], next_cursor: string | null, has_more: boolean, total_in_batch: number }
- SwipeRecord: { productId: string, action: SwipeAction, timestamp: number } — for undo stack
- PendingFeedback: { productId: string, action: SwipeAction, retryCount: number }

**stores/swipeStore.ts:**
Create Zustand store (NO persist — feed state is session-only, not localStorage):
- State:
  - cards: FeedItem[] — current card queue
  - currentIndex: number — position in queue
  - undoStack: SwipeRecord[] — last 5 actions for undo
  - pendingFeedback: PendingFeedback[] — failed submissions awaiting retry
  - isLoading: boolean
  - error: string | null
  - nextCursor: string | null — for pagination
  - hasMore: boolean
- Actions:
  - setCards(items: FeedItem[], nextCursor: string | null, hasMore: boolean) — set initial/appended cards
  - appendCards(items: FeedItem[], nextCursor: string | null, hasMore: boolean) — append next batch
  - advanceCard(action: SwipeAction) — increment currentIndex, push to undoStack (max 5)
  - undoLastSwipe() -> SwipeRecord | null — pop from undoStack, decrement currentIndex
  - addPendingFeedback(feedback: PendingFeedback) — queue failed submission
  - removePendingFeedback(productId: string) — remove after successful retry
  - setLoading(loading: boolean)
  - setError(error: string | null)
  - reset() — clear all state (on logout or page leave)
- Computed (via getters or separate selectors):
  - currentCard: cards[currentIndex] or null
  - remainingCards: cards.length - currentIndex
  - canUndo: undoStack.length > 0

Follow onboardingStore pattern for structure but do NOT use persist middleware — feed data should not persist across sessions.

**services/feedbackService.ts:**
Create API service functions using the useApi pattern (import axios instance or create one with auth):
- fetchFeed(cursor?: string, pageSize?: number): Promise&lt;FeedResponse&gt; — GET /api/feed with query params
- submitFeedback(productId: string, action: SwipeAction): Promise&lt;void&gt; — POST /api/feedback with { product_id, action }

Use the existing Axios instance pattern from the project (check how useApi.ts or authStore exports the configured client). The feedback service should be a plain module with exported async functions, NOT a hook — hooks will compose these.

**hooks/useFeed.ts:**
Create useFeed hook that manages feed lifecycle:
- On mount: fetch initial feed batch (pageSize=20), call swipeStore.setCards()
- Prefetch trigger: when remainingCards <= 5 AND hasMore AND NOT isLoading, fetch next batch with nextCursor, call swipeStore.appendCards()
- Image prefetch: when cards are loaded, prefetch images for next 3 cards using `new Image().src = url` pattern (from CalibrationStep)
- Return: { currentCard, remainingCards, isLoading, error, hasMore, refetch }
- Use useEffect for initial fetch and useEffect with remainingCards dependency for prefetch trigger
- Handle errors: set store error state, don't crash

Do NOT use TanStack Query — keep it simple with useEffect + store pattern matching the existing codebase style.
