  <files>apps/stylipp.com/src/features/onboarding/stores/onboardingStore.ts, apps/stylipp.com/src/features/onboarding/components/OnboardingLayout.tsx, apps/stylipp.com/src/App.tsx</files>
  <action>
1. Create features/onboarding/stores/onboardingStore.ts — Zustand with persist:
   - State:
     - currentStep: number (0-2: photos, calibrate, profile)
     - photoKeys: string[] — S3 keys of uploaded photos
     - photoEmbeddings: number[][] — 768-dim vectors from backend
     - calibrationItems: string[] — product IDs shown during calibration
     - calibrationLikes: string[] — product IDs liked
     - calibrationDislikes: string[] — product IDs disliked
     - isComplete: boolean
   - Actions:
     - addPhoto(key: string, embedding: number[]): push to photoKeys + photoEmbeddings
     - removePhoto(index: number): splice from both arrays
     - setCalibrationItems(ids: string[]): store item IDs
     - addLike(productId: string): push to calibrationLikes
     - addDislike(productId: string): push to calibrationDislikes
     - complete(): set isComplete=true
     - reset(): clear all state back to initial
   - Persist config:
     - name: 'stylipp-onboarding'
     - storage: createJSONStorage(() => localStorage)
     - partialize: persist photoKeys, photoEmbeddings, calibrationLikes, calibrationDislikes, currentStep (NOT calibrationItems — refetch on resume)

2. Create features/onboarding/components/OnboardingLayout.tsx:
   - Full-height flex column layout (minHeight: 100dvh)
   - Top: optional step title/description
   - Middle (flex: 1): step content via `<Outlet />` wrapped in framer-motion AnimatePresence
   - Bottom: MUI MobileStepper with dots variant, 3 steps
   - Track direction (forward/back) for slide animation direction
   - Slide variants: enter from right (forward) or left (back), exit opposite
   - Transition: tween, easeInOut, 0.3s duration
   - Next/Back buttons: navigate to appropriate step route
   - Disable Back on step 0, disable Next when step requirements not met (delegate to each step's "Continue" button instead — stepper just shows progress)

3. Update App.tsx:
   - Replace onboarding placeholder with nested routes:
   ```tsx
   <Route element={<PrivateRoute />}>
     <Route path="/onboarding" element={<OnboardingLayout />}>
       <Route index element={<Navigate to="photos" replace />} />
       <Route path="photos" element={<PhotoUploadStep />} />
       <Route path="calibrate" element={<CalibrationStep />} />
       <Route path="profile" element={<ProfileStep />} />
       <Route path="complete" element={<OnboardingComplete />} />
     </Route>
     <Route path="/feed" element={<div>Feed (Phase 5)</div>} />
   ```

   Import OnboardingLayout. Use lazy placeholder components for CalibrationStep, ProfileStep, OnboardingComplete (simple "Coming soon" divs for now — built in Plan 05).

**Avoid:** Don't put Next/Back navigation logic in MobileStepper buttons — each step has its own "Continue" button that validates before advancing. MobileStepper only shows dot progress. Don't use React Router's useBlocker to prevent back-navigation during onboarding — it's confusing UX. Users should be able to go back.

<task type="auto">
  <name>Task 2: Build photo upload step with compression and preview</name>
  <files>apps/stylipp.com/src/features/onboarding/components/PhotoUploadStep.tsx, apps/stylipp.com/src/features/onboarding/hooks/usePhotoUpload.ts</files>
  <action>
1. Create features/onboarding/hooks/usePhotoUpload.ts:
   - `usePhotoUpload()` hook returns: { uploadPhoto, isUploading, progress, error }
   - `uploadPhoto(file: File)` async function:
     - Set isUploading=true, progress=0
     - Compress with browser-image-compression: { maxSizeMB: 1, maxWidthOrHeight: 1920, useWebWorker: true }
     - Track compression progress as 0-50% of total
     - Create FormData with compressed file
     - POST to /api/onboarding/photos with Axios, track upload progress as 50-100%
     - On success: return { key, url, embedding } from response
     - On error: set error message, return null
     - Always set isUploading=false at end

2. Create features/onboarding/components/PhotoUploadStep.tsx:
   - Heading: "Upload Your Outfits" with subtitle "Share 1-2 photos of outfits you love"
   - Two photo slots side by side (MUI Grid or flexbox, each ~45% width):
     - Slot 1: "Outfit 1" (required)
     - Slot 2: "Outfit 2" (optional)
   - Each slot:
     - Empty state: dashed border box with camera/add icon, clickable
     - Hidden `<input type="file" accept="image/*">` (NO capture attribute — per iOS research)
     - On file select: show preview immediately with URL.createObjectURL(file), start upload
     - Uploading state: preview image with CircularProgress overlay showing percentage
     - Complete state: preview image with small checkmark badge
     - Error state: preview with error icon, tap to retry
     - Remove button (small X icon) to clear and re-upload
   - "Continue" button at bottom:
     - Disabled until at least 1 photo uploaded successfully (embedding received)
     - On click: store photo data in onboarding store via addPhoto(), navigate to /onboarding/calibrate
   - Use MUI: Box, Typography, Card, CardMedia, IconButton, CircularProgress, Button
   - Clean up object URLs on unmount (URL.revokeObjectURL in useEffect cleanup)

**Avoid:** Don't use `<input capture="environment">` — broken on iOS PWA standalone mode (WebKit bug #206219 per research). Don't use FileReader.readAsDataURL for previews — use URL.createObjectURL() (faster, less memory). Don't store File/Blob objects in Zustand persist — only store S3 keys and embeddings (serializable). Don't require 2 photos — 1 is minimum, 2 is preferred but optional.
  </action>
