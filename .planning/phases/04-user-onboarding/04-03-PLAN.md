  <files>apps/stylipp.com/package.json, apps/stylipp.com/src/shared/hooks/useApi.ts, apps/stylipp.com/src/features/auth/stores/authStore.ts</files>
  <action>
1. Install dependencies in apps/stylipp.com:
   ```bash
   cd apps/stylipp.com && pnpm add zustand browser-image-compression react-hook-form @hookform/resolvers zod
   ```
   Check if axios is already installed (it's in PROJECT.md stack). If not: `pnpm add axios`.

2. Create shared/hooks/useApi.ts — Axios instance:
   - Base URL: `/api` (Vite proxy handles forwarding to backend)
   - Request interceptor: attach `Authorization: Bearer {token}` from auth store (if token exists)
   - Response interceptor: on 401, clear auth state and redirect to /login
   - Export the configured axios instance as default

3. Create features/auth/stores/authStore.ts — Zustand with persist:
   - State: token (string | null), user ({ id, email, display_name, onboarding_completed } | null)
   - Actions:
     - `login(email: string, password: string)`: POST /api/auth/login, store token + user
     - `register(email: string, password: string, displayName?: string)`: POST /api/auth/register, store token + user
     - `logout()`: clear token + user, navigate to /login
     - `fetchMe()`: GET /api/auth/me, update user state (for refreshing onboarding status)
   - Persist config: name 'stylipp-auth', partialize to only persist token (fetch user on rehydration via fetchMe)

4. Create directory structure:
   ```
   src/features/auth/
   ├── stores/
   │   └── authStore.ts
   └── components/
       ├── LoginPage.tsx    (next task)
       └── RegisterPage.tsx (next task)
   src/shared/
   ├── hooks/
   │   └── useApi.ts
   └── components/
       └── PrivateRoute.tsx (next task)
   ```

**Avoid:** Don't store the full user object in localStorage persist — only persist the token, then fetch user info on rehydration. This ensures onboarding_completed is always fresh. Don't import from auth store inside useApi.ts at module level (circular dep risk) — use getState() inside the interceptor.

<task type="auto">
  <name>Task 2: Create login/register pages and PrivateRoute guard</name>
  <files>apps/stylipp.com/src/features/auth/components/LoginPage.tsx, apps/stylipp.com/src/features/auth/components/RegisterPage.tsx, apps/stylipp.com/src/shared/components/PrivateRoute.tsx, apps/stylipp.com/src/App.tsx</files>
  <action>
1. Create shared/components/PrivateRoute.tsx:
   - Check auth store for token
   - If no token: `<Navigate to="/login" replace />`
   - If token: `<Outlet />`
   - Simple wrapper, no loading state needed (token check is synchronous from Zustand)

2. Create features/auth/components/RegisterPage.tsx:
   - MUI Container (maxWidth="xs") centered on page
   - Typography heading: "Create Account" (use i18n key if available)
   - React Hook Form with Zod schema: email (EmailStr), password (min 8 chars), confirmPassword (must match)
   - MUI TextField for each field with error display from form state
   - Submit button calls authStore.register()
   - On success: navigate to /onboarding
   - On 409 error: show "Email already registered" error
   - Link to /login at bottom: "Already have an account? Sign in"

3. Create features/auth/components/LoginPage.tsx:
   - Same layout pattern as RegisterPage
   - React Hook Form + Zod: email, password
   - Submit calls authStore.login()
   - On success: check user.onboarding_completed — if true navigate to /feed, if false navigate to /onboarding
   - On 401: show "Invalid email or password" error
   - Link to /register: "Don't have an account? Sign up"

4. Update App.tsx routes:
   ```tsx
   <Routes>
     {/* Public routes */}
     <Route path="/" element={<HomePage />} />
     <Route path="/login" element={<LoginPage />} />
     <Route path="/register" element={<RegisterPage />} />

     {/* Protected routes */}
     <Route element={<PrivateRoute />}>
       <Route path="/onboarding/*" element={<div>Onboarding (Plan 04-04)</div>} />
       <Route path="/feed" element={<div>Feed (Phase 5)</div>} />
     </Route>

     <Route path="*" element={<NotFoundPage />} />
   </Routes>
   ```
   Use placeholder divs for /onboarding and /feed — they'll be built in subsequent plans.

**Avoid:** Don't over-style the auth pages — simple, clean MUI components are sufficient for MVP. Don't add social auth (OAuth) — email/password only per PROJECT.md. Don't add "forgot password" flow — out of scope for MVP. Don't use controlled MUI TextFields with React Hook Form (use register() for uncontrolled — better performance).
  </action>
