  <action>
1. Add `passlib[bcrypt]` to pyproject.toml dependencies.

2. In auth/schemas/schemas.py:
   - Add `RegisterRequest(BaseModel)`: email (EmailStr), password (str, min_length=8), display_name (str | None = None)
   - Update `UserResponse` to include: id, email, display_name, onboarding_completed, created_at
   - Add `AuthResponse(BaseModel)`: access_token (str), token_type (str = "bearer"), user (UserResponse)

3. In auth/service/service.py:
   - Import passlib CryptContext with bcrypt scheme
   - `hash_password(password: str) -> str` using CryptContext
   - `verify_password(plain: str, hashed: str) -> bool` using CryptContext
   - `async def register_user(session: AsyncSession, email: str, password: str, display_name: str | None) -> User` — check email uniqueness (raise 409 if exists), hash password, create User, return it
   - `async def authenticate_user(session: AsyncSession, email: str, password: str) -> User | None` — query by email, verify password, return User or None

4. In auth/router/router.py:
   - `POST /auth/register` — validate input, call register_user, create JWT with create_access_token(user.id), return AuthResponse with user info + token. Return 409 on duplicate email.
   - Fix `POST /auth/login` — replace 501 placeholder. Call authenticate_user, return 401 on failure, return AuthResponse with JWT on success.
   - Fix `GET /auth/me` — query full User from DB using user_id from JWT dependency, return UserResponse (not just user_id dict). This lets frontend check onboarding_completed status.
   - All DB operations use `async with AsyncSession` from database module.

**Avoid:** Don't use synchronous bcrypt calls — passlib's CryptContext.hash/verify are CPU-bound but brief enough for MVP. Don't create a separate users repository yet — keep queries in service for now (simple CRUD).
  </action>
  <verify>

- Same register again returns 409
- `curl -X POST /api/auth/login -H "Content-Type: application/json" -d '{"email":"test@test.com","password":"test1234"}'` returns 200 with access_token
- `curl -H "Authorization: Bearer {token}" /api/auth/me` returns full user with onboarding_completed=false
  </verify>
