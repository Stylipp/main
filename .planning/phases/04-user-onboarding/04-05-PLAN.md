  <action>
1. Create features/onboarding/hooks/useCalibration.ts:
   - On mount (useEffect):
     - Get photoEmbeddings from onboarding store
     - POST /api/onboarding/calibration-items with { embeddings: photoEmbeddings }
     - Store returned items in local state (array of CalibrationItem objects with product_id, title, price, currency, image_url)
     - Also store product IDs in onboarding store via setCalibrationItems()
   - State: items (CalibrationItem[]), currentIndex (number), isLoading (boolean), error (string | null)
   - Actions:
     - `like()`: add current item's product_id to onboardingStore.calibrationLikes, advance currentIndex
     - `dislike()`: add to onboardingStore.calibrationDislikes, advance currentIndex
   - Computed: currentItem, progress (currentIndex / total), isComplete (currentIndex >= items.length)
   - Handle error states: if API fails, show retry option

2. Create features/onboarding/components/CalibrationStep.tsx:
   - Heading: "Discover Your Style" with subtitle "Like or pass on these items (X of 15)"
   - Loading state: MUI Skeleton cards while fetching calibration items
   - Main interaction area (centered, ~80% width):
     - Product card: MUI Card with CardMedia (product image, aspect-ratio ~3:4), title below, price below title
     - Card should be prominent — this is the focus of the screen
     - Subtle entry animation (framer-motion: fade + slight scale) for each new card
   - Action buttons below card (horizontal row, centered):
     - Left: IconButton with ThumbDown icon (red/error color), onClick → dislike()
     - Right: IconButton with ThumbUp icon (green/success color), onClick → like()
     - Buttons should be large (48px+) for easy mobile tapping
   - Progress: MUI LinearProgress at top showing currentIndex/total, or text "X of 15"
   - On isComplete: auto-navigate to /onboarding/profile
   - Handle empty/error: if no items returned, show error message with retry button

**Avoid:** Don't implement swipe gestures — that's Phase 6 (custom swipe component with framer-motion gesture physics). Keep this as simple button-based interaction. Don't preload all images — just load current + next (prefetch next image via new Image() for smooth transitions). Don't show price in a way that biases the calibration — show it small below the title, not prominently.

  <files>apps/stylipp.com/src/features/onboarding/components/ProfileStep.tsx, apps/stylipp.com/src/features/onboarding/components/OnboardingComplete.tsx, apps/stylipp.com/src/shared/components/OnboardingGate.tsx, apps/stylipp.com/src/App.tsx</files>
  <action>
1. Create features/onboarding/components/ProfileStep.tsx:
   - Heading: "Almost There!" with subtitle "Add your name (optional)"
   - Simple form:
     - MUI TextField for display_name (optional, max 100 chars)
     - No other fields for MVP — keep it minimal to reduce friction
   - "Complete Setup" button (MUI Button, variant="contained", full width):
     - On click: set loading state, call POST /api/onboarding/complete with:
       - photo_embeddings: from onboarding store
       - liked_product_ids: from onboarding store
       - disliked_product_ids: from onboarding store
     - On success: call authStore.fetchMe() to refresh user (now onboarding_completed=true), call onboardingStore.complete(), navigate to /onboarding/complete
     - On error: show error message, allow retry
   - Show loading spinner on button during API call (backend computes user vector + price profile — may take a few seconds)

2. Create features/onboarding/components/OnboardingComplete.tsx:
   - Celebration screen with:
     - Large checkmark icon or celebration illustration (MUI CheckCircle icon, large, primary color)
     - Heading: "You're All Set!"
     - Subtitle: "We've learned your style. Your personalized feed is ready."
     - "Start Discovering" button → navigate to /feed
   - Clear onboarding store on mount (onboardingStore.reset()) — no longer needed
   - Simple, clean, encouraging

3. Create shared/components/OnboardingGate.tsx:
   - Get user from auth store
   - If !user: render nothing (PrivateRoute handles auth redirect)
   - If user && !user.onboarding_completed: `<Navigate to="/onboarding" replace />`
   - If user && user.onboarding_completed: `<Outlet />`

4. Update App.tsx route structure:
   ```tsx
   <Routes>
     {/* Public */}
     <Route path="/" element={<HomePage />} />
     <Route path="/login" element={<LoginPage />} />
     <Route path="/register" element={<RegisterPage />} />

     {/* Auth required, onboarding NOT required */}
     <Route element={<PrivateRoute />}>
       <Route path="/onboarding" element={<OnboardingLayout />}>
         <Route index element={<Navigate to="photos" replace />} />
         <Route path="photos" element={<PhotoUploadStep />} />
         <Route path="calibrate" element={<CalibrationStep />} />
         <Route path="profile" element={<ProfileStep />} />
         <Route path="complete" element={<OnboardingComplete />} />
       </Route>
     </Route>

     {/* Auth + onboarding required */}
     <Route element={<PrivateRoute />}>
       <Route element={<OnboardingGate />}>
         <Route path="/feed" element={<div>Feed coming in Phase 5</div>} />
       </Route>
     </Route>

     <Route path="*" element={<NotFoundPage />} />
   </Routes>
   ```

   Replace all placeholder components from Plan 04-04 with actual imports.

**Avoid:** Don't add more profile fields (age, gender, size) — keep MVP minimal. Don't skip the completion screen (it provides positive feedback). Don't forget to call authStore.fetchMe() after completion — the auth store needs the updated onboarding_completed=true to pass OnboardingGate.
