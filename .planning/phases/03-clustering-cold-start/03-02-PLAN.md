  <action>
1. Create `schemas/schemas.py` with Pydantic models:
   - `ClusterInfo`: cluster_index (int), product_count (int), prior_probability (float)
   - `ClusterStatsResponse`: total_clusters (int), total_products (int), clusters (list[ClusterInfo]), silhouette_score (float | None)
   - `ColdStartRequest`: embeddings (list[list[float]]) — 1-2 photo embeddings from user uploads
   - `ColdStartMatch`: product_id (str), score (float), cluster_index (int), is_diversity (bool)
   - `ColdStartResponse`: matches (list[ColdStartMatch]), primary_clusters (list[int]), diversity_clusters (list[int]), total_matches (int)
   - `RebuildResponse`: k (int), silhouette_score (float), product_count (int), cluster_sizes (dict[int, int])

2. Create `schemas/__init__.py` exporting all schemas.

3. Create `cold_start_service.py` with ColdStartService class:
   - `__init__(self, qdrant_client, settings)` — accepts async Qdrant client and settings

   - `async def find_nearest_clusters(self, embeddings: list[list[float]], top_k: int = 5) -> list[dict]`:
     Average the input embeddings (if multiple photos, compute mean vector).
     Search "style_clusters" Qdrant collection for top_k nearest centroids by cosine similarity.
     Return list of {cluster_index, score, product_count} dicts sorted by score descending.

   - `async def get_products_by_cluster(self, cluster_index: int, limit: int = 20) -> list[dict]`:
     Search "products" Qdrant collection filtered by payload.cluster_id == cluster_index.
     Return product points with their payloads and scores.
     Use qdrant_client.scroll() with filter for exact match on cluster_id.

   - `async def get_cold_start_feed(self, embeddings: list[list[float]], feed_size: int = 20) -> ColdStartResponse`:
     Orchestrator method:
     a) Find top 3 nearest clusters (primary clusters).
     b) Find 2 adjacent clusters (clusters ranked 4th-5th by similarity — these are "diversity" clusters).
     c) Allocate feed: 17 items from primary clusters (proportional to similarity score), 3 items from diversity clusters (mandatory per PROJECT.md: 3/20 diversity injection).
     d) For each cluster, use scroll with cluster_id filter to get products. Sort by Qdrant score if available, otherwise random.
     e) Build ColdStartResponse with is_diversity flag on diversity items.

   Diversity injection is MANDATORY (per PROJECT.md). Do NOT make it optional or configurable — always inject 3/20 items from adjacent clusters to prevent echo chambers.
