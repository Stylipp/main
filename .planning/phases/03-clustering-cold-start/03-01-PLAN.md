<task type="auto">
  <name>Task 2: Create ClusteringService with K-means and optimal k determination</name>
  <files>apps/backend/src/features/clustering/service/clustering_service.py, apps/backend/src/features/clustering/service/cluster_repository.py, apps/backend/pyproject.toml</files>
  <action>
1. Add `scikit-learn>=1.4.0` to `apps/backend/pyproject.toml` dependencies.

2. Create `cluster_repository.py` with ClusterRepository class:
   - `get_all_clusters(session) -> list[StyleCluster]`
   - `upsert_clusters(session, clusters: list[dict]) -> int` — bulk upsert cluster metadata (delete old + insert new)
   - `get_cluster_by_index(session, cluster_index: int) -> StyleCluster | None`

3. Create `clustering_service.py` with ClusteringService class:
   - `__init__(self, qdrant_client, settings)` — accepts async Qdrant client and settings

   - `async def fetch_all_embeddings(self) -> tuple[list[str], np.ndarray]`:
     Scroll ALL product points from the "products" Qdrant collection.
     Return (point_ids, embeddings_matrix) as numpy array.
     Use scroll with limit=100, iterating until no more points.
     This is the input to K-means.

   - `def find_optimal_k(self, embeddings: np.ndarray) -> int`:
     Test k values from settings.min_clusters to min(settings.max_clusters, n_samples // 2).
     Use silhouette_score from sklearn.metrics for each k.
     Return k with highest silhouette score.
     If n_samples < min_clusters * 2, use n_samples // 5 as min k (handle small catalogs).
     Log silhouette scores for each k tested.

   - `def run_kmeans(self, embeddings: np.ndarray, k: int) -> tuple[np.ndarray, np.ndarray]`:
     Run sklearn.cluster.KMeans with n_clusters=k, random_state=42, n_init=10.
     Return (centroids, labels) — centroids shape (k, 768), labels shape (n_samples,).

   - `async def store_results(self, point_ids, labels, centroids, session)`:
     a) Upsert cluster metadata to PostgreSQL via ClusterRepository (cluster_index, product_count, prior_probability for each cluster).
     b) Upsert centroid vectors to Qdrant "style_clusters" collection — each point has id=cluster_index, vector=centroid, payload={cluster_index, product_count, prior_probability}.
     c) Update product points in Qdrant "products" collection — set payload.cluster_id for each product point based on its cluster label. Use qdrant_client.set_payload() in batches.

   - `async def rebuild_clusters(self, session) -> dict`:
     Orchestrator method: fetch_all_embeddings → find_optimal_k → run_kmeans → store_results.
     Return stats dict: {k, silhouette_score, product_count, cluster_sizes}.

Use asyncio.to_thread for CPU-intensive operations (find_optimal_k, run_kmeans) to avoid blocking the event loop. Do NOT run K-means directly in async context.
  </action>
