     - Dependency: get_db session, get_qdrant_client
     - Instantiate ClusteringService, call rebuild_clusters(session)
     - Return RebuildResponse with stats
     - This is a long-running operation (~10-60s depending on catalog size), so set a generous timeout

   - `GET /clustering/stats` — Get cluster statistics:
     - Dependency: get_db session
     - Query all StyleCluster records via ClusterRepository
     - Return ClusterStatsResponse with cluster count, product count, and cluster info list

   - `POST /clustering/cold-start` — Get cold start recommendations:
     - Accept ColdStartRequest body (list of embeddings)
     - Dependency: get_qdrant_client
     - Validate: embeddings must be non-empty, each embedding must be 768-dimensional
     - Instantiate ColdStartService, call get_cold_start_feed(embeddings)
     - Return ColdStartResponse

   - `async def get_clustering_service(...)` — wires Qdrant client + settings into ClusteringService

Follow the established dependency injection pattern from products router (see 02-05 patterns).

  <name>Task 2: Wire clustering into app lifespan and entrypoint</name>

  <action>
1. Update `main.py`:
   - Import clustering router and include it: `app.include_router(clustering_router, prefix="/api")`
   - In the lifespan context manager, after embedding service loads:
     a) Call `ensure_cluster_collection()` from core/qdrant.py to create the "style_clusters" collection if it doesn't exist
     b) Log "Cluster collection ready"
   - Do NOT run clustering automatically in lifespan (it's CPU-intensive and blocks startup). Clustering is triggered via API endpoint or entrypoint script.

2. Update `entrypoint.sh`:
   - AFTER the existing auto-seeding logic (which checks product count and seeds if empty):
   - Add auto-clustering logic:
     ```
     # Auto-cluster if no clusters exist
     CLUSTER_COUNT=$(python -c "
     import asyncio
     from src.core.qdrant import get_qdrant_client
     from src.core.config import get_settings
     async def count():
         client = await get_qdrant_client()
         settings = get_settings()
         try:
             info = await client.get_collection(settings.cluster_collection)
             return info.points_count
         except Exception:
             return 0
     print(asyncio.run(count()))
     " 2>/dev/null || echo "0")

     if [ "$CLUSTER_COUNT" -eq "0" ] || [ "$CLUSTER_COUNT" = "0" ]; then
         echo "No clusters found. Running initial clustering..."
         python -m scripts.rebuild_clusters
         echo "Initial clustering complete."
     else
         echo "Found $CLUSTER_COUNT clusters. Skipping clustering."
     fi
     ```
   - This follows the same idempotent pattern as product seeding (check count, act if empty, skip otherwise)

IMPORTANT: The entrypoint clustering MUST run AFTER seeding (products need to exist before clustering). Maintain the existing order: migrations → seeding → clustering → uvicorn.
  </action>
