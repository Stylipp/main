  <files>apps/backend/src/features/products/service/ingestion_service.py</files>
  <action>
1. In `ingest_or_update_product`, after looking up the existing product and before calling `_update_existing`, check if the product is archived (`existing.archived_at is not None`).

2. If archived, clear the archive status:
   - Set `existing.archived_at = None` on the SQLAlchemy model
   - Call `qdrant.set_payload()` to set `{"archived": False}` on the Qdrant point (use `str(existing.id)` as point ID)
   - Log: `"Un-archiving returning product %s/%s", product_data.store_id, product_data.external_id`

3. Then proceed with the normal `_update_existing` flow to update metadata/re-embed as needed.

This prevents duplicate creation when a product is removed from a sitemap then re-listed later. The existing `external_id + store_id` lookup already finds the archived record — we just need to clear the archive flag.

Do NOT change the lookup logic in `get_by_external_id` — it should find products regardless of archived status (no WHERE archived_at IS NULL filter on lookup).
  </action>
