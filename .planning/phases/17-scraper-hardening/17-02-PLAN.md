  <name>Task 2: Add archive/unarchive service methods + archive endpoint</name>

  <action>
1. Add `BatchArchiveRequest` schema to schemas.py:
   ```python
   class BatchArchiveRequest(BaseModel):
       store_id: str
       external_ids: list[str]
   ```

2. Add `BatchArchiveResponse` schema:
   ```python
   class BatchArchiveResponse(BaseModel):
       archived_count: int
       archived_ids: list[str]
   ```

3. Add `archive_products` method to IngestionService:
   - Accept `external_ids: list[str]` and `store_id: str`
   - Query PostgreSQL for Product records matching (external_id IN external_ids AND store_id = store_id AND archived_at IS NULL)
   - Set `archived_at = datetime.utcnow()` on matched records
   - For each archived product, call `qdrant.set_payload()` to add `{"archived": True}` to the Qdrant point. Use `str(product.id)` as the point ID (same format used in ingestion).
   - Return list of archived external_ids
   - Log count of archived products

4. Add `POST /api/products/archive/batch` endpoint to router.py:
   - Accept `BatchArchiveRequest` body
   - Call `service.archive_products(request.external_ids, request.store_id)`
   - Return `BatchArchiveResponse`
   - Wire up dependencies same as `ingest_batch` (needs IngestionService)

Do NOT hard-delete from Qdrant — use `set_payload` to flag, not `delete`. This preserves interaction history.
  </action>
