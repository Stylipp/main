1. Add a `_normalize_category_value` helper function (or update the existing tokenization logic) that normalizes apostrophe-like characters BEFORE token matching:
   ```python
   import unicodedata

   def _normalize_text(value: str) -> str:
       decoded = unquote_plus(str(value)).strip().lower()
       decoded = unicodedata.normalize("NFC", decoded)
       # Normalize Hebrew geresh and all apostrophe variants to ASCII
       decoded = decoded.replace("\u05F3", "'")  # Hebrew geresh ׳
       decoded = decoded.replace("\u2019", "'")  # Right single quote '
       decoded = decoded.replace("\u02BC", "'")  # Modifier letter apostrophe ʼ
       decoded = decoded.replace("\u2018", "'")  # Left single quote '
       return decoded
   ```

2. Apply this normalization in `normalize_raw_categories` BEFORE splitting into tokens and matching against keyword sets. The current code already does `unquote_plus` and `lower()` — integrate the apostrophe normalization into that flow.

3. Verify existing Hebrew keywords already cover both apostrophe forms. The current keyword set has:
   - ג'ינס and ג׳ינס (jeans) — already covered
   - קפוצ'ון and קפוצ׳ון (hoodie) — already covered
   Check for any keywords that use geresh but are MISSING the ASCII apostrophe variant (or vice versa). With the normalization in place, only the ASCII form needs to be in the keyword set.
